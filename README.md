# Architecture-Hyperparameter-Research-CIFAR-10

### Research Goal
This project aims to develop a convolutional neural network (CNN) that accurately classifies images from the CIFAR-10 dataset into their respective categories. A convolutional neural network is a type of deep learning model specifically designed for processing grid-structured data such as images. Unlike traditional fully connected neural networks, CNNs use convolutional layers to automatically learn spatial hierarchies of features. Examples are edges, textures, and object parts, which are all learned through the application of learnable filters. These filters slide across the input image to detect meaningful patterns while preserving spatial relationships, making CNNs particularly effective for image classification tasks.
This study will systematically explore the impact of hyperparameter tuning on model performance, with the objective of identifying an architecture that maximizes classification accuracy while maintaining computational efficiency. In particular, we will examine architectural and training choices such as the number of convolutional layers, kernel sizes, pooling strategies, activation functions, learning rates, batch sizes, and the number of training epochs.
We plan to compare both computational efficiency and model performance between Neural Architecture Search (NAS) and Differentiable Architecture Search (DARTS) to tune the network architecture, as well as Bayesian Optimization (BO) methods for hyperparameter tuning. By evaluating how these design decisions influence learning behavior and predictive performance, the project seeks to provide a clear justification for each selected parameter in the final model. The results of these experiments will allow us to draw conclusions explaining why certain hyperparameter configurations lead to improved performance, thereby demonstrating a principled and systematic approach to CNN design for image classification tasks.

### Planned Approaches
To begin our research, architecture search and hyperparameter optimization were separated into two distinct stages. Although both influence model performance, they operate at different levels of the learning process. Architecture defines the structural capacity of the model, while hyperparameters govern how that structure is trained. Separating architecture search from hyperparameter tuning allows for a more controlled and systematic optimization process, reducing confounding effects and improving computational efficiency. By first identifying a strong architectural configuration and then refining its training dynamics, we can better attribute performance gains and manage resource constraints.
We first selected Neural Architecture Search (NAS) as the foundational framework for this project in order to systematically automate the design of convolutional neural networks (CNNs). Rather than manually specifying architectural choices, NAS allows us to explore combinations of structural components, including the number of convolutional layers, filter sizes, kernel dimensions, activation functions, and pooling strategies, within a predefined search space. Each sampled architecture is evaluated using validation performance, enabling objective comparison across configurations. While both grid search and random search can be used to explore this space, we initially implemented random search due to limited computational resources. Random search enables broad exploration while significantly reducing computational cost compared to exhaustive grid search, which becomes infeasible as the search space expands.
To further improve search efficiency and scalability, we also incorporate Differentiable Architecture Search (DARTS), a gradient-based extension of NAS. Unlike traditional discrete NAS methods that require training separate models for each candidate architecture, DARTS relaxes architectural choices into a continuous space by representing operations as weighted mixtures. Using a bilevel optimization framework, network weights are optimized with respect to training loss, while architectural parameters are optimized with respect to validation loss. This simultaneous optimization of weights and architecture can reduce computational requirements while maintaining strong predictive performance. DARTS therefore provides a more efficient and scalable alternative for architecture optimization within our study.
Beyond architectural design, hyperparameter optimization is also critical to maximizing model performance, and our research emphasizes Bayesian Optimization as an efficient and principled approach to hyperparameter tuning. Bayesian Optimization models validation performance as a probabilistic function and constructs a surrogate model often based on a Gaussian process to approximate the relationship between hyperparameters and performance. An acquisition function then selects new configurations by balancing exploration of uncertain regions with exploitation of promising areas. This approach enables the identification of high-performing hyperparameter configurations with fewer evaluations than traditional methods. Together, NAS via random search and DARTS, and Bayesian Optimization provide a structured and computationally efficient framework for optimizing CNN architectures within practical resource constraints.
Overall, the architecture search will focus on convolutional depth, kernel size, pooling type, and activation functions, while Bayesian Optimization will be applied to tune key training hyperparameters such as learning rate and batch size. This combined strategy aims to systematically and efficiently identify high-performing neural network configurations while minimizing computational cost.

#### Performance Optimization Plan
The performance optimization process begins with an exploration of the CNN architecture prior to hyperparameter tuning. The initial focus is on identifying an effective structural configuration using Neural Architecture Search (NAS). The search space is deliberately constrained to ensure computational feasibility while still allowing meaningful architectural variation. Specifically, the search space includes modifications to the number of convolutional layers, kernel sizes, pooling types, and activation functions. The number of convolutional layers is varied between two and five layers; configurations exceeding five layers are excluded due to the application of pooling at every layer, which would excessively reduce spatial dimensions. Kernel sizes are limited to 3×3 and 5×5 filters, while pooling operations include both max pooling and average pooling. Activation functions considered in the search space are ReLU and Leaky ReLU. Our search space was intentionally designed based on architectural and optimization strategies commonly explored in research literature, reflecting well-established design choices in prior CNN studies.
As a subset of NAS, Differentiable Architecture Search (DARTS) is applied to further refine architectural selection, specifically focusing on pooling type, activation functions, and kernel dimensions as the number of layers and filter sizes cannot be altered during DARTS runtime. Using gradient-based optimization, architectural parameters are updated iteratively alongside network weights. Gradients are evaluated at each step to determine which architectural components are preferred, enabling efficient identification of high-performing configurations without exhaustively retraining separate models.
After establishing a strong architectural foundation, hyperparameter optimization is conducted using Bayesian Optimization. This stage focuses specifically on tuning the learning rate and batch size. By modeling the performance landscape probabilistically and selecting hyperparameter configurations based on expected improvement, Bayesian Optimization enables efficient and systematic refinement of the training process.
